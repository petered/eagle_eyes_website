Mapping, CalTopo & Sessions

5.1 Map Interface

The map panel on the Fly Screen shows the drone's position, flight path, detections, and other operational data overlaid on map imagery. Eagle Eyes uses OSMDroid for map rendering with support for multiple tile sources.

Map Gestures

GestureAction
DragPan the map
PinchZoom in/out
Double-tapZoom to tapped point
Two-finger rotateRotate the map orientation

Tile Sources

You can change the map background imagery depending on what is most useful for your operation:

SourceDescriptionWhen to Use
Esri World ImageryHigh-resolution satellite imageryMost SAR operations — see actual terrain, buildings, roads, vegetation
OpenStreetMapDetailed street and trail mapUrban search, finding access routes, identifying trails and structures
OpenTopoTopographic map with contour linesMountainous terrain, elevation-aware planning
USGS TopoUS Geological Survey topographic mapUS operations requiring official USGS data

The map displays several types of markers that update in real time during flight:

Tap any detection marker on the map to see its thumbnail, confidence score, and timestamp. This is a quick way to review what was found in a specific area without opening the full Detection Viewer.

5.2 Offline Maps

SAR operations often take place in areas with limited or no cellular connectivity. Eagle Eyes supports MBTiles files for offline map use, ensuring you have map imagery even without internet.

Loading an Offline Map

  1. Go to ☰ Menu → Map Loader
  2. Tap Select MBTiles
  3. Choose your .mbtiles file from storage
  4. The map tiles load and are available for offline use
⚠️
Download offline maps before heading to the field. MBTiles files can be large (hundreds of MB for detailed coverage of a large area). Prepare these in advance when you have a good internet connection.

5.3 Connecting to CalTopo

CalTopo is a powerful mapping platform widely used in SAR. Eagle Eyes integrates directly with CalTopo, allowing you to see team assignments, search areas, and markers on your drone's map — and push detections back to CalTopo for the whole team to see.

Method 1: QR Code (Fastest)

  1. Generate QR code on the CalTopo website
  2. In Eagle Eyes: ☰ Menu → CalTopo Connection
  3. Tap Scan QR Code
  4. Point the device camera at the QR code

Method 2: Manual Entry

  1. ☰ Menu → CalTopo Connection
  2. Tap Enter Map ID
  3. Type the CalTopo map ID
  4. Enter credentials and connect

Connection Status

StatusMeaning
🟢 GreenConnected and syncing
🟡 YellowConnected but sync delayed
🔴 RedDisconnected

5.4 What Syncs

From CalTopo → Eagle Eyes

  • Markers and points
  • Lines and routes
  • Polygons and search areas
  • Team member positions

From Eagle Eyes → CalTopo

  • Manual markers
  • Detection locations
  • Detection images

This bidirectional sync means your incident commander can draw search areas on CalTopo and they immediately appear on your drone's map. When the AI detects something, the detection location and thumbnail are pushed back to CalTopo where the entire team can see it.

5.5 Service Account Setup New in 1.6.0

Service accounts allow automatic map connections without scanning QR codes each time. This is ideal for dedicated SAR tablets that stay in team vehicles.

What is a Service Account?

  • A persistent CalTopo connection credential
  • Configured once by the team lead
  • Automatically reconnects to the team map on app startup
  • No QR scanning needed in the field

Setting Up a Service Account

  1. In CalTopo, create a service account for your organization
  2. Generate the service account credentials
  3. In Eagle Eyes: ☰ Menu → CalTopo Connection
  4. Tap Service Account
  5. Enter the service account credentials
  6. Select your team's default map
  7. Tap Save

Auto Re-Sync

With a service account configured:

  • Connection restores automatically after app restart
  • Sync resumes when network becomes available
  • No manual intervention needed in the field
💡
Service accounts are especially useful for tablets that stay in team vehicles. Configure once and the connection persists across operations.

5.6 Sessions

A session in Eagle Eyes is a container that groups together all the data from a single mission or operation. Every detection, map feature, and configuration setting is tied to the active session. Understanding sessions helps you keep your operations organized and your data separated.

What a Session Stores

Each session contains:

  • All AI detections — Every bounding box, confidence score, thumbnail, and GPS coordinate captured during that session
  • Manual annotations — Any detections you marked manually
  • Map features — GeoJSON features (lines, polygons, markers) created or synced during the session
  • CalTopo connection — Which CalTopo map this session is linked to, and whether auto-connect is enabled
  • Map visibility filters — Which map layers and features are shown/hidden
  • Mission-specific configuration — UI preferences and settings tied to this session

How Sessions Are Created

Eagle Eyes uses an intelligent auto-reuse system for sessions:

  • When you open the app, it checks the last session's timestamp
  • If the last session was opened less than 12 hours ago, the app automatically resumes that session — you pick up right where you left off
  • If more than 12 hours have passed, the app treats it as a new mission and creates a new session automatically
  • You can also manually create a new session at any time through the Session Manager

This means if you close the app for a lunch break and reopen it an hour later, you're still in the same session and all your detections stay together. But if you come back the next day for a new operation, a fresh session starts automatically.

Session Manager

Access the Session Manager via ☰ Menu → Sessions. From here you can:

  • View all sessions — Listed with name and creation date, newest first. The active session is highlighted.
  • Create a new session — Give it a name (e.g., "Morning Search - North Ridge") and optional description. The new session becomes active immediately.
  • Rename a session — Tap the edit icon to change the name at any time.
  • Switch sessions — Tap a session to make it the active one. All subsequent detections and data will be recorded under this session.
  • Delete a session — Removes the session and all its associated detections, map features, and configuration. This cannot be undone.

Session Identification

Each session has a unique ID generated using ULID (Universally Unique Lexicographically Sortable Identifier). In the UI, sessions are displayed as a short identifier plus the name you gave it:

*3K2: Morning Search - North Ridge (2026-02-22 08:30)

  • * indicates this is the currently active session
  • 3K2 is the last 3 characters of the unique ID
  • Morning Search - North Ridge is the name you assigned
  • (2026-02-22 08:30) is the creation timestamp

Why Sessions Matter for SAR

In a real SAR operation, you may run multiple flights over several days. Sessions let you:

  • Keep each day's or each sortie's detections separate so you can review them independently
  • Link a session to a specific CalTopo map for that operation
  • Filter detections by session in the Detection Viewer (see Module 3)
  • Delete test/training data without affecting real mission data
  • Share or reference a specific session's findings in reports
💡
Best practice: Name your sessions descriptively at the start of each operation. Something like "Feb 22 AM - River Valley East" is much easier to find later than "Session 1".
