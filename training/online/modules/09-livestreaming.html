Module 6

Broadcasting & Livestreaming

⏱️ 35 minutes

Learning Objectives

  • Understand the three telemetry broadcast types and when to use each
  • Configure telemetry broadcasting for your operation
  • Set up a livestream to command centers
  • Share the stream URL with viewers
  • Manage streaming while flying

6.1 Telemetry Broadcast Options

The Telemetry Broadcast Card on the home page shows three broadcast types that control how your drone's position and telemetry data is shared with others. Each type serves a different audience and use case.

Broadcast TypeProtocolAudienceBest For
LocalUDPSame WiFi networkClose-range team coordination
MapMQTTCalTopo map viewersRemote team monitoring
PublicEDSBWorldwideMulti-agency SAR operations

Local Broadcast (UDP)

  • Shares position with other Eagle Eyes drones on the same WiFi network
  • Direct peer-to-peer on UDP port 7742 at 10 Hz — no server needed
  • Other drones appear as markers on your map showing position, heading, altitude, and battery
  • Best for close-range team coordination where all operators are on the same local network

Map Broadcast (MQTT to CalTopo)

  • Shares your drone's position to your connected CalTopo map
  • Visible to anyone viewing the CalTopo map — including remote command centers
  • Requires an active CalTopo connection (see Module 5)
  • Works over cellular or any internet connection — operators don't need to be on the same network

Public Broadcast (EDSB)

  • Shares position to the Eagle Eyes global drone map
  • Visible to all EDSB users worldwide
  • Optional public drone ID for identification (e.g., "SAR-Team-Alpha-1")
  • QR code links directly to your drone's public position
  • Ideal for large-scale, multi-agency SAR where teams aren't on the same network or CalTopo map

Configuring Broadcasts

  1. Tap the Telemetry Broadcast card on the home page
  2. Toggle each broadcast type ON/OFF independently:
    • Local — WiFi network sharing
    • Map — CalTopo map sharing
    • Public — EDSB global sharing
  3. For EDSB, configure your Public Drone ID so other teams can identify you
⚠️
Public broadcasts are visible worldwide. Only share information you're comfortable making public. Use Local or Map broadcast for operations that don't need global visibility.
💡
You can enable multiple broadcast types simultaneously. For example, enable Local for nearby team coordination AND Map for remote command center monitoring at the same time.

6.2 Livestream Overview

Eagle Eyes can livestream the drone's video feed in real time to any web browser. This lets command centers, incident commanders, or remote team members watch what the drone sees without needing any special software.

FeatureCapability
TechnologyWebRTC (peer-to-peer video)
Latency1-3 seconds
ViewersMultiple simultaneous
Quality480p / 720p / 1080p

6.3 Starting a Stream

  1. Go to ☰ Menu → Livestream
  2. Configure resolution and bitrate
  3. Tap Start Stream
  4. Wait for connection (green indicator)
  5. Copy the stream URL
  6. Share the URL with viewers

Viewers open the URL in any web browser — no app needed.

💡
Test streaming before the mission. Pre-share the URL with the command center so they can bookmark it and be ready when you launch.

6.5 Bandwidth Requirements

ResolutionMinimum Upload Speed
480p1 Mbps
720p2.5 Mbps
1080p5+ Mbps
⚠️
In remote areas with limited cellular coverage, use 480p to maintain a stable stream. A dropped or stuttering stream is worse than a lower resolution one.
