Module 6

Multi-Drone Coordination

⏱️ 60 minutes

Learning Objectives

  • Configure UDP telemetry broadcasting
  • View other Eagle Eyes drones on your map
  • Coordinate multiple drones during SAR operations

8.1 How It Works

Eagle Eyes uses UDP broadcasting to share drone positions:

  • Each drone broadcasts telemetry every 100ms
  • All drones on the same network receive broadcasts
  • Other drones appear as markers on your map
  • No server required - direct peer-to-peer
SettingValue
PortUDP 7742
Broadcast255.255.255.255
Update Rate10 Hz

8.2 Enabling Broadcasting

  1. Open Quick Status Dialog
  2. Tap Network card
  3. Toggle UDP Telemetry Broadcast ON

8.3 Viewing Other Drones

When teammates broadcast, their drones appear on your map showing:

  • Position (real-time location)
  • Heading (direction arrow)
  • Altitude and battery (in info window)
  • Device name/call sign

8.5 Telemetry Broadcast Options New in 1.6.0

The Telemetry Broadcast Card on the home page shows three broadcast types:

Local Broadcast (UDP)

  • Shares position with drones on the same WiFi network
  • Direct peer-to-peer, no server needed
  • Best for close-range team coordination

Map Broadcast (MQTT to CalTopo)

  • Shares position to your connected CalTopo map
  • Visible to anyone viewing the CalTopo map
  • Requires CalTopo connection

Public Broadcast (EDSB)

  • Shares position to the Eagle Eyes global drone map
  • Visible to all EDSB users worldwide
  • Optional public drone ID for identification
  • Great for large-scale, multi-agency SAR coordination
💡
Use EDSB for multi-agency SAR where teams aren't on the same WiFi network.

Configuring Broadcasts

  1. Tap Telemetry Broadcast card on home page
  2. Toggle each broadcast type ON/OFF:
    • Local - WiFi network sharing
    • Map - CalTopo map sharing
    • Public - EDSB global sharing
  3. Configure Public Drone ID for EDSB (optional)

Public Drone ID

For EDSB public tracking:

  • Set a recognizable ID (e.g., "SAR-Team-Alpha-1")
  • Other teams can see your drone on the global map
  • QR code links directly to your drone's public position

8.7 Custom MQTT Topics

For advanced Map broadcast configuration, you can set custom MQTT topics:

When to Use Custom Topics

  • Multiple teams using the same CalTopo map
  • Separating drone traffic by function (search, rescue, logistics)
  • Creating private channels for specific operations

Configuring Custom Topics

  1. Open Settings → Telemetry
  2. Tap MQTT Topic ID
  3. Enter a 10-character topic identifier
  4. Share the topic ID with your team
ℹ️
Topic IDs are limited to 10 characters. Use consistent naming like "SAR-ALPHA1" across your team.

8.8 Contact Info Privacy

Control what information is shared with your telemetry broadcasts:

Configuring Privacy Settings

  1. Open Settings → Privacy
  2. Configure what to share:
    • Device Name - Your drone/tablet identifier
    • Operator Name - Your name or call sign
    • Contact Info - Phone/email (optional)
Broadcast TypeRecommended Privacy
Local (UDP)Full info - team members only
Map (MQTT)Name + call sign
Public (EDSB)Call sign only - minimal info
⚠️
Public broadcasts are visible worldwide. Only share information you're comfortable making public.
