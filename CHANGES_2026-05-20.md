# Frontend changes — 2026-05-20

Builds on yesterday's `CHANGES_2026-05-19.md`. Backend answered the four open questions from the deferred list; this commit acts on the parts that don't need further backend coordination.

All changes on `adminpage` branch.

## What shipped

### 1. Suppress Renew button on already-renewed licenses

**File**: `account.html`

Backend's new `license_overview` shape now carries `successor_license_id` + `successor_starts_at_iso` (shipped in their PR #49 per yesterday's reply). The card renderer now uses both:

- When `license.successor_license_id` is set, the per-card Renew button is replaced with a passive **"Renewal queued"** status badge showing the start date ("Starts October 7, 2027"). The badge uses the same dimensions as the action button so the card layout doesn't reflow.
- The top-of-section yellow-highlight renewal notice now also skips licenses with a queued renewal — they're already taken care of, no need to nag.
- A small helper `formatRenewalStartDate(iso)` renders the ISO start date as a human-readable date with a graceful fallback.

### 2. Per-card Renew → purchase page (no more mailto)

**File**: `account.html`

The "Renew for 1 year" button on each license card now deep-links to `/account/purchase/?mode=renew&license_id=X` instead of opening a mailto. The purchase page picks up the URL params, preselects renew mode, and selects the target license in the picker.

### 3. Full `renew_license` endpoint integration

**File**: `purchase.html`

The renew-mode markup that's been sitting drafted on the page since the IA restructure is now fully wired.

**Mode availability** (`applyAvailableModes`):
- If the active account has any active license: "New license" mode is hidden, "Renew an existing license" mode is shown.
- If the account also has at least one license eligible to renew (active + no queued successor): the Renew radio is enabled.
- If no active licenses: "New license" shows as before, Renew is hidden.
- Falls back gracefully if the URL preselects a mode that's now hidden.

**Renew picker** (`renderRenewLicenseList`):
- Lists every active license without a queued successor.
- One-line label per license: name + device count + expiry + "renewal will run for 1 year starting at expiry".
- Same `.license-row` visual as the add-devices picker (`data-renew-license-id` distinguishes the click handler).

**Continue dispatcher** branches by `state.mode === 'renew'`:
- **Payment = credit card** (`onBuyNow`): POSTs `/renew_license` with `{ license_id, payment_method: 'checkout' }`, redirects to the returned `hosted_url` (Stripe Checkout).
- **Payment = invoice** (`onInvoice`): POSTs `/renew_license` with `{ license_id, payment_method: 'invoice', custom_fields }`, opens the returned `hosted_url` in a new tab.
- **Payment = quote** (`onQuote`): POSTs `/renew_license` with `{ license_id, payment_method: 'quote' }` — **omits `custom_fields`** because the backend explicitly rejects them on quote-mode renewals (Stripe's API limitation on one-time-priced Quote line items). Opens hosted_url in a new tab.

**409 billing_address_required handling**: extracted into a shared `postWithAddressRetry(endpoint, body, reason)` helper used by both renewal and self-service invoice/quote paths. Same modal pattern as yesterday.

**Order summary** (`recomputeSummary`): renew mode shows `Renew "X" — N devices, 1 year` with the per-year price (the existing `priceForNew(n)` rate; backend final-prices via the catalog product_id).

**Cart summary** (`cartSummaryLines`): renew mode emits a 2-line entry for invoice/quote bodies and email fallbacks.

**accountScopeChanged**: switching active accounts now re-evaluates `applyAvailableModes`, re-renders both license pickers, and clears `state.renewLicenseId` so the new account's licenses get picked up cleanly.
